# Global Instructions

## Auto-apply skills by language

When working with **Golang** projects, automatically apply these skills:
- golang-project-layout
- golang-code-style
- golang-naming
- golang-error-handling
- golang-testing
- golang-design-patterns
- golang-concurrency
- golang-safety
- golang-security
- golang-performance
- golang-linter

When working with **Python** projects, automatically apply:
- python-patterns
- python-testing
- django-patterns (if Django)

When working with **Rust** projects, automatically apply:
- rust-patterns
- rust-testing

When working with **Kotlin** projects, automatically apply:
- kotlin-patterns
- kotlin-testing

When working with **Java** projects, automatically apply:
- java-coding-standards
- springboot-patterns (if Spring Boot)

When working with **TypeScript/JavaScript** projects, automatically apply:
- frontend-patterns
- nextjs-turbopack (if Next.js)
- bun-runtime (if Bun)

When working with **Docker/DevOps**, automatically apply:
- docker-patterns
- deployment-patterns

## General
- Always apply security-review skill when touching auth, API, or user input
- Always apply api-design skill when creating REST/gRPC endpoints
- Always apply database-migrations skill when modifying database schemas
